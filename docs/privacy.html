Firsttry Governance for Jira – Privacy Policy

Last updated: 2025-12-22

Who We Are

Legal Entity: Arnab Poddar

Contact Email: security@firsttry.run

FirstTry Governance is an Atlassian Forge app designed for governance monitoring and audit trail capture in Jira environments. This app operates entirely within the Atlassian Forge platform security sandbox.

What Data the App Accesses

FirstTry Governance accesses the following configuration metadata only from your Jira instance in read-only mode:

  • Jira configuration metadata: Project configurations, issue type schemas, field definitions, workflow status values, automation rule metadata
  • Audit-relevant metadata: Configuration timestamps, schema structure, field mappings
  • System metadata: Available projects, fields, and statuses (schema-level information only)

Access Pattern: All data access is read-only. The app captures configuration state for governance monitoring purposes only.

What Data the App Does NOT Collect

FirstTry Governance explicitly does NOT collect, access, or process:

  • No Personally Identifiable Information (PII): No user emails, names, profile information, or personal data
  • No User Content: No issue descriptions, comments, attachments, or work logs
  • No Credentials: No passwords, API tokens, or authentication secrets
  • No Personal Profiles: No user preferences, avatars, or account details
  • No Behavioral Tracking: No analytics, usage tracking, or behavioral profiling

The app operates on configuration metadata only, with no access to issue content or user-generated data.

Where Data is Processed and Stored

Processing Location: All data processing occurs within the Atlassian Forge platform security sandbox.

Storage Location: Data is stored exclusively in tenant-isolated Forge Storage, managed by Atlassian.

Infrastructure: Atlassian-managed infrastructure only. No external servers or third-party hosting.

Tenant Isolation: All stored data is isolated per Jira tenant. No cross-tenant data access or sharing occurs.

Platform Security: Data security, encryption, and access control are managed by the Atlassian Forge platform. The app delegates authentication, authorization, and encryption to Atlassian's platform security mechanisms.

Third-Party Data Sharing

None.

FirstTry Governance has zero external egress. The app does not:

  • Share data with any third-party services
  • Make external API calls outside the Atlassian Forge platform
  • Transmit data to external servers or analytics services
  • Use external logging, monitoring, or tracking services

All network communication is limited to Atlassian Forge platform APIs only. This is enforced by Forge platform egress controls and the app's current permission configuration.

Data Retention

Retention Period: Data stored in Atlassian Forge hosted storage follows Atlassian's Forge data lifecycle. Upon app uninstallation, hosted storage data is not immediately purged; it is retained for a limited period per Atlassian platform behavior and may be recoverable for a short window. After the platform retention period, hosted storage data is deleted.

Data Export: Tenant administrators can export all stored data via the app's admin interface (JSON format).

Security Practices

FirstTry Governance leverages Atlassian Forge platform security mechanisms:

  • Platform Sandboxing: App executes in Atlassian's isolated Forge runtime environment
  • No External Egress: Zero outbound network connections to external services (verified by automated testing)
  • Tenant Isolation: Data is isolated per Jira tenant using Forge Storage scoping
  • Atlassian-Managed Encryption: Encryption at rest and in transit managed by Atlassian Forge platform
  • Platform Authentication: User authentication and authorization delegated to Atlassian platform
  • Read-Only Access: App uses read-only Jira API access (no write permissions)

Security Disclosure: The app does not hold independent security certifications (SOC 2, ISO 27001). Security posture inherits from the Atlassian Forge platform.

Vulnerability Reporting: To report security vulnerabilities, use GitHub Security Advisory.

Your Rights

Depending on your jurisdiction, you may have the following rights:

  • Right to Access: Export your data via the app's admin interface (JSON export)
  • Right to Deletion: Uninstall the app; hosted storage data will be deleted after the platform retention period
  • Right to Portability: Data export is available in machine-readable JSON format

Note: As a Forge app, data processing is governed by Atlassian's Data Processing Agreement (DPA). Refer to Atlassian's privacy policies for platform-level data handling.

Changes to This Policy
